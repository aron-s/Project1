Docker-

Docker is a set of platform as a service products that use OS-level virtualization to deliver software in packages called containers. Some terminologies of Git are given below.


Containers

  • A container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another. A Docker container image is a lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, system libraries and settings.

Images

  • Docker images are the basis of containers. It is the package with all the dependencies and information needed to create a container. An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime. Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container's filesystem. An image is immutable once it has been created.

Dockerhub

  • Dockerhub is a hosted repository service provided by Docker for finding and sharing container images. Click here to view this project on dockerhub.

Some basic docker commands-

These are the basic commands for docker that I know-
  • "docker â€“version": This command displays the current version of docker installed.
  • "docker ps":Shows list of currently running containers.
  • "docker build": Builds an image from a dockerfile.
  • "docker images": Displays all docker images.
  • "docker run": Create a container of an image.
  • "docker stop": Stop a running container.
  • "docker rm": Remove a stopped container. The "-f" argument can stop and remove a running container.
  • "docker compose up --build": Build a container from a docker-compose.yml file.
  • "docker commit": Create a new image of an edited container on the local system.
  • "docker push": Push image into docker hub repository.
  • "docker pull": Pull an image from a docker repository.
  • "docker login": Log into docker hub repository.
  • "docker tag": Retag a local image with a new image name and tag.

How to start this project and access the container using docker

  1. View my project on my docker repository.
  2. In a terminal, use the "docker pull as4259/project1:master" command to pull the image into the local scope.
  3. Use the "docker run -p -it 80:80 as4259/project1:master" command to run and access the container.
  4. Once done in the interactive terminal, use the "docker rm -f as4259/project1" command to stop and remove the container.
