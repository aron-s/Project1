Python/Flask-

Flask is a micro web framework written in Python. It is classified as a microframework because it does not require particular tools or libraries. This project was created using flask. How python/flask was used in this project is documented below.


Pytest

Pytest is a testing framework based on python. For this project, pytest was used to make test cases for the different pages of the website and ensure every one of them worked as intended. The different cases tests the contents in each web page and also weather they return an 200 OK status code.
A snippet of code below shows an example of a test case used-

def test_request_index(client):
  """This makes the index page"""
  response = client.get("/")
  assert response.status_code == 200
  assert b"Welcome to my Website" in response.data

The code above asserts that in our index page we have the words "Welcome to my Website" and that the web page is returning a 200 OK status code. When creating simple pages, after editing/adding the contents of the page, I updated the test cases accordingly to make sure that the website works as updated.


Creating Webpages for the Project

All of the webpages in the project are inside the simple_pages folder. We also have a base template page for the project. which contains the navigation bar and the footer. All of the pages inside simple_pages are created extending the base html template using jinja notation and flask. To create the simple_pages bootstrap 5 was used and pytest as mentioned above was used to test them out.

File structure of the Project

This project is structured as shown below-
  • Project Root Folder <- Root of the project
    • .dockerignore -> List files to ignore that are not needed to create an image
    • .gitignore -> List files to ignore for commits to git
    • docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
    • Dockerfile <-File used to create an image to run in a container that runs the program
    • hekoru.yml <- Contains configuration to host the image on heroku
    • Readme <- Contains information displayed at the github repository of the project
    • requirements.txt <- requirements text file
    • setup.py <- setup tools
    • .github - a folder
      • workflows - Folder containing github workflows routines
        • dev.yml <- Contains workflow configurations to run for the development version. It runs when there is a pull request.
        • prod.yml - Contains workflow configurations to run for the production version. It runs when commits are pushed or merged into the master branch.
    • app -a Folder containing app utilities
      • context_processors - creates a namespace
        • __init__.py -> Contains python code for few utility functions.
      • simple_pages -> a folder
        • templates -> Folder containing the web pages
          • about.html -> About Me page
          • index.html -> The index/home page
          • page1.html -> Page containing Git information
          • page2.html -> Page containing Docker information
          • page3.html -> Page containing python/flask information
          • page4.html -> Page containing CI/CD information
        • __init_.py -> Contains flask code to render web pages.
      • static -> a folder
        • css -> Folder containing css files
          • styles.css -> Contains CSS styling for webpages
        • icons -> Folder containing icons
          • facebook.png - Facevook logo
          • Github-Mark-Light-32px.png - Github logo
          • instagram.png -> Instagram logo
          • linkedin.png -> LinkedIn logo
          • Logo 32_32 px.png -> A logo for title
        • images -> Folder containing images
          • cicd.png -> CI/CD image
          • flask.png -> Flask image
          • Git-Logo-2Color.png -> Git image
          • homepage-docker-logo.png -> Docker image
          • mypic1.jpg -> Image of myself
          • sample.jpg -> Sample image
        • js -> Folder containing javascript files
          • scripts.js -> Javascript file containing javascripts used in the project.
        • templates -> a folder containing base template
          • base.html -> Base html template use
        • __init__.py -> Creates flask app
        • run.py -> This allows Gunicorn to serve the app in production
    • tests -> Folder containing test routines
      • __init__.py -> pytest file
      • calculator_test.py -> test for calculator program (for future)
      • conftest.py -> pytest file to test if the application is loading
      • context_process_test.py -> tests the utility context processes
      • simple_pages_test.py -> pytest file to test the html pages
