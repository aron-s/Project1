AAA Testing-

The AAA testing is a pattern used in unit testing that has become a standard in the industry. It suggests that you should divide your test method into three sections: Arrange, Act and Assert.


Tutorial for Arrange, Act, Assert

Arrange-Act-Assert is a great way to structure test cases. It prescribes an order of operations:

  1. Arrange inputs and targets: Arrange steps should set up the test case. Does the test require any objects or special settings? Does it need to prep a database? Does it need to log into a web app? Handle all of these operations at the start of the test.
  2. Act on the target behavior: Act steps should cover the main thing to be tested. This could be calling a function or method, calling a REST API, or interacting with a web page. Keep actions focused on the target behavior.
  3. Assert expected outcomes: Act steps should elicit some sort of response. Assert steps verify the goodness or badness of that response. Sometimes, assertions are as simple as checking numeric or string values. Other times, they may require checking multiple facets of a system. Assertions will ultimately determine if the test passes or fails.
To provide an example, we will create a simple test using this pattern in python using a calculator class. The calculator class contains a property "result" that stores the result from a calculation.

Step 3: Act upon the system under test.

The example above creates a test case and asserts that the class property "result" is working as expected.

Why Testing is Important

Software Testing is important because it ensures that the product delivered is defective free which in turn emphasizes higher quality and proves to be cost-effective. Bugs (programming errors) can cause all sorts of problems for users. Testing helps to uncover these bugs so that they can be fixed before the software is released to the public. Testing can also help to ensure the quality of the software by identifying and reporting any defects or inconsistencies in the functionality of the software. In other words, it checks to make sure that the software does what it's supposed to do.
A few other reasons as to why software testing is important-
  • Increase the quality: Testing enables you to see what the software does and how well it does it so that the business can measure the quality of the software before it goes live.
  • We all make mistakes: There is no such thing as a defect-free system, and we all make mistakes especially developing a system that is complex.
  • Reduce risks: Defects must be sought out and either fixed or removed from the final product to ensure that the system can run as expected during live operation.
  • Cost Effective: It is often more time consuming and difficult to fix a defect, hence more expensive, when a defect is discovered at a later stage and a large amount of the system has already been built.
  • Any bugs are found easily and quicker
  • Testing saves time and money
  • Reusable and Reliable
